## Saving
Obsidian handles saving on its own, don't worry about it

## Git usage from Obsidian
- Ctrl + P to open the obsidian command bar
- Type "Git"
	1. Git: pull
		- Downloads all commits (this prevents any merge conflicts before they get messy)
		- ![[Pasted image 20260614160748.png]]
	2. Git: commit
		- Rounds up all local changes into a "commit"
		- ![[Pasted image 20260614160609.png]]
	3. Git: push
		- Uploads all commits to "origin" (the online repository)
		- ![[Pasted image 20260614160626.png]]

>[!warning]
>Talk to eachother, avoid editing the same files 😭
## Handling of conflicts
Ideally, just don't edit files at the same time, communication does wonders, no system is fail proof

If your local version and the online version ("origin" or "remote" in Obsidian) have discrepancies within a file, a merge conflict will happen, same with doing pull requests if enabled

[Github Desktop](https://desktop.github.com/download/) can fix most conflicts, otherwise use console commands or cry
